## Prototype
- クラスからインスタンスを作るのではなく、既存のインスタンスからコピーして新しいインスタンスをつくるパターン

### メリット
- 以下のように単純にクラスからインスタンスを生成できない場合に役立つ
  - 扱うオブジェクトの種類が多すぎて一つのクラスとしてまとめられない場合
  - クラスからのインスタンス生成が複雑で難しい場合 ex. GUIでつくる画像のインスタンス
  - インスタンスを生成するときのフレームワークを、特定のクラスに依存しないようにつくりたい場合

### java.lang.Cloneableインタフェース
- Java言語仕様で定義された、自分自身を複製するメソッドであるclone()を実行する際に、実装する必要のあるインタフェース
  - Cloneableインタフェースを実装したクラスのインスタンスがclone()メソッドを呼び出すと、コピーされたインスタンスを返す
  - Cloneableインタフェースの中身はない、ただclone()によってコピーすることができるというマーカーとして使われる
    - マーカーインタフェース
    - clone()メソッド自体はjava.lang.Objectクラスで定義されている
- clone()メソッドは浅いコピーをする
  - コピーしたいインスタンスのフィールドに参照型があるときに注意
  - 浅いコピーで困る場合は、clone()をオーバーライドすることもできる
