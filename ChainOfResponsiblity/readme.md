 ## Chain of Responsibility
 - ある要求が発生したときに、その処理するオブジェクトをダイレクトに決められない場合、鎖状につないだオブジェクトを渡り歩いてオブジェクトを決定するパターン

### メリット
- たらい回すことで個々のクラスが自分の仕事の集中できる（できることが明確になる）
